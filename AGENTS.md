## ⚙️ Code Style

### Go Conventions
- Follow **Effective Go**.
- Run `go fmt` before committing.
- Keep functions short (prefer < 50 lines).
- Use **table-driven tests**; coverage target > 80%.

### Naming
- Exported: `PascalCase` (types, functions, constants)
- Unexported: `camelCase`
- Interface names end with `-er` when applicable (e.g. `Reader`)
- Domain enums use `PascalCase` with Chinese comments (e.g. `DengMing MonthGeneral = iota // 登明（亥）`)

### Imports
- Group: stdlib → third-party → project internal
- No dot imports or blank imports unless required

### Error Handling
- Return errors explicitly; do not swallow them.
- Error messages: **Traditional Chinese** with English technical terms where appropriate.
- Use `fmt.Errorf` for context wrapping.

### Comments
- All exported symbols must have a comment.
- Use `// SymbolName ...` format.
- Example:
  ```go
  // Calculate 執行大六壬排盤計算
  func (e *Engine) Calculate(req DivinationRequest) (*DivinationResult, error)
  ```

## 🚫 Anti-Patterns

| Pattern | Why Forbidden |
|---------|---------------|
| Hardcoding calendar data | Use `lunar-zenith` service or `LocalDataSource` |
| Modifying generated proto types directly | Update contract / `.proto` first, then regenerate |
| Skipping contract validation | Run `make verify-contracts` before commit |
| Mixing REST/gRPC logic | Keep handlers separate (`internal/httpapi/` vs `pkg/server/`) |
| Hand-editing `.env.ports` | It is generated by `make sync-contracts` |
| Using solar terms for month general | Month general changes at mid-terms (中氣), not solar terms |
| Using day/night for tianjiang direction | Direction depends on noble's earth-plate position |
| Using stem-only for xunKong | Must use full day pillar (60-Jia cycle) |

## 📋 Contract-First Rules

1. **All API changes** must follow `contracts/openapi/liuren-zenith.yaml` (symlink to `destiny-cloud/contracts`).
2. Update the contract **before** modifying code.
3. Do not add fields or change types that are not defined in the contract.
4. CI enforces `make verify-contracts` on every PR.
